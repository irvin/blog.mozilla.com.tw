整合 Firebug 與開發者工具

你可能已經聽聞我們正努力整合原生的 Firefox 開發者工具 (DevTools) 以及 Firebug。除了要將 Firebug 所有的高人氣功能移植到開發者工具之外，也要讓雙方的多程序 (Multi-process) 相容，還能用於遠端除錯 (如 e10s 相容)。我們也更努力要讓 Firebug 的移植過程盡可能簡單且順利。是的，如之前所提過的，我們就是要提供單一的好用工具給開發者！

來看看目前的進度。

下一代 Firebug 的目標，就是要讓使用者能自在的搭配使用開發者工具。這也是 Firebug 3 (也就是 Firebug.next) 的初衷。Firebug 3 並不是另外一種開發者工具，而是披在開發者工具之上的輕薄外衣，讓開發者工具看起來就像 Firebug。另有幾項附加功能，也將由我們逐步移植到開發者工具之中。

如果你想續用任何功能 (可能是 Firebug 稍早版本的功能)，也可等著這些功能回鍋到開發者工具中。

Firebug 3

如果你安裝 Firebug 3 (第一次 Beta 試用)。就來看看下方擷圖所呈現的開發者工具外觀。看來很眼熟嗎？


AMO 尚未全面發佈 Firebug 3 (也就是 Firebug.next)，因此你可能必須透過 about:config，先將 xpinstall.signatures.required 設為「false」。

這就是將 Firebug 主題移植到 DevTools 所帶來的一部分 UI 與配置優點。而 Firefox 工具列中也提供大家所熟知的 Firebug 開始鈕，可藉此進入開發者工具箱。

看到了嗎？現正啟動了「Firebug Theme」選項。

想必大家都很期待 Firebug 與正宗 Firefox 開發者工具的整合。小組也正努力要將兩者密切統整，其能為開發者帶來更方便、更強大、更好用的工具。若你同是此兩款開發工具的愛用者，就請繼續參閱原文以了解更多移植細節。
