概述 UP 使用者個人化

Mozilla Labs 團隊最近一篇文章談到 Firefox 新的個人化自主權，而這個概念也和 Jay Sullivan 在 5 月份撰文闡述的「Personalization with Respect」不謀而合。我們希望每個人都能更深入了解自己在 Web 上的互動行為，再透過清楚定義的方式而輕鬆取得自己需要的東西。目前已有領導業界的出版商與行銷商支持這個想法，同時樂見能為客戶提供更豐富、更完整、更吸引人的內容。

Mozilla 所提出的使用者個人化 (User Personalization，UP) 與許多創新概念，就是要提供更高品質的使用者經驗。為了達到此目的，消費者可於互動過程中選擇是否揭露某種類型的資訊，以及資訊揭露的程度，進而取得 Web 上最高相關性的內容與服務。最近的著名範例，就是英國衛報 (The Guardian) 讓閱報者選擇是否關閉其網站上有關皇室寶寶的新聞。如此讓閱報者能根據自己個人喜好而確實參與內容交換的機制。

Mozilla 必須坦承，沒有任何東西是真正「免費」的。任何精采內容都需要「成本」，而不會像變魔術一樣憑空出現。廠商可透過廣告、訂戶、創新經營模式，或其他使用者互動形式來打平這些成本。雜亂無趣的內容同樣也有成本，但會降低使用者瀏覽意願、廣告展示價格 (CPM)，甚至網站的整體價值。若使用經驗不友善又缺乏透明度，除了會減少網站流量之外，也會影響利害關係人對網站的評價。

針對這個生態系統，我們希望各階層的內容建構者與服務供應商都能雨露均霑。行銷大師 Jon Bond 對此下了最佳註解：「出版商與知名品牌往往根據使用者經驗所創造的價值，直接評斷網路所產生的價值，進而強化 Web 所提升的經濟效益。」

而目前的難題在於，使用者不清楚所需考量的優缺點，而且沒有多少人真正了解免費內容與服務背後的價值主張 (Value proposition)。因為現有系統缺乏透明性，更造成「個人資料易遭惡意利用」的印象，進而傷害了消費者對出版商、知名品牌、服務供應商的信心。Web 上真正有價值的是信任感；而非 CPM，也不是曝光度，更不是網站流量。

Mozilla 所追求的個人化，就是要協助消費者能適時取得最高相關度的內容，且過程完全攤在陽光下，務必讓大家能自由放心選擇。

為了測試此議題，Mozilla 與不同的利害關係人進行會談，包含使用者、出版商、開發者、知名品牌、隱私權倡導者、廣告技術供應商等。根據會談結果，廠商與使用者均給予了正面肯定，並承諾會積極加入這個前景可期的生態系統，讓相關意圖和興趣的資訊更趨完備。

主要德文出版商 Hubert Burda Media 的首席科學家 Jean-Paul Schmetz 就說：「Mozilla 所提議的 interests API 讓我們了解，必先尊重使用者的隱私與選擇，才能讓使用者真正顯現出自己的興趣。我們也很高興能夠透過 Burda 的網站來驗證此論點。」

著名的 LUMAscape 創辦人 Terry Kawaja 也建議：「interests API 能根據消費者喜好而提供自主選擇權，所建構的生態系統甚至可能替代消費者在不自覺情況下所選擇的隱性追蹤生態系統。相同條件的數位供應生態，都能透過 interests API 取得消費者的興趣與意圖，進而創造公平的競爭環境。」

曾為 IAB 前主席，現為 TheDailyMeal.com 與 TheActiveTimes.com 的發行人 Jim Spanfeller 表示：「從許多方面來看，interests API 可針對目前內容與經驗個人化的概念，朝著更重視消費者安全的方向發展，並協助內容供應商針對使用者的興趣與需求，進一步強化網頁的呈現效果，為使用者提供絕佳體驗。」

EFF 主任技師 Dan Auerbach 也同樣對此一理念給予正面評價：「Mozilla 的 UP 專案，就是以使用者的隱私為本，重新設計 Web 上的個人化機制。類似專案都可視為使用者隱私的一大進展。」

雖然這個概念備受肯定，但仍有很長的路要走。如 Mozilla Labs 文章中所提到，UP 目前仍處於實驗階段，並等著測試許多假設性的條件。後續測試將讓我們更了解完整機制，有助我們隨時做出相關調整。總的來說，Mozilla 認為這個充滿未來性的解決方案，可同時滿足個人與出版商的需求，並提供更好、更透明、更有價值的 Web 經驗。
