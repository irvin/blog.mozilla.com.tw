W3C 宣佈 HTML5 進入「建議」階段

HTML5 在今天列入了 W3C 的「建議 (Recommendation)」公告；也就是成為 W3C 標準前的最後一個階段。Mozilla 身為創新開發 HTML5 的諸多組織之一，深入耕耘 HTML5 的標準化已久。而 W3C 在今天所宣佈的內容，正好呼應了 Mozilla 最近十年所念茲在茲的使命與心血。


圖片來源：http://commons.wikimedia.org/

Mozilla 另已率先廣泛採用多項技術 (如 WebGL)，除了能進一步強化並提升 HTML5 的競爭力之外，也希望 HTML5 有機會能替代原生與私有的生態系統。隨著 Firefox OS 切入智慧型手機的市場，Mozilla 也在行動 Web 領域大有斬獲。而透過 Firefox OS 努力推廣的許多新 API 與功能，目前均已進入標準化的程序。之前只有原生 App 才具備的功能，未來都能在 Web 上順利運作。

成為 W3C 的標準 (Standard) 之前，須完成如企畫提案 (Proposal)、編輯草案 (Editors’ Draft)、候選建議 (Candidate Recommendation)，直到最後的建議 (Recommendation) 等一系列階段。而「建議」階段是極重要的里程碑。Mozilla 鼓勵開發者能在新的 Web 標準真正塵埃落定之前，就儘早開始接觸。若想跟上目前潮流，Web 開發者應留意新標準，且從「編輯草案」就應隨時緊盯相關動向，而不要進入「建議」階段之後才開始投入心力。另外如 developer.mozilla.org 與 caniuse.com 等網站，均是以 Web 開發者為目標族群來提供相關文章，亦可作為絕佳的資訊取得管道。

Mozilla 針對 HTML5 所著重的第二個重要領域，就是測試集 (Test suite)。Web 開發者甚或 Web 引擎等類似性質的開發者，均可透過測試集檢驗 Web 瀏覽器是否貫徹 HTML5 的規格。你可透過下列連結觀看最新結果：

http://w3c.github.io/test-results/dom/all.html

http://w3c.github.io/test-results/html/details.html

在確保使用者獲得一致 Web 經驗的作業上，這些 HTML5 自動化測試集的確扮演了重要角色。

Mozilla 本就放眼於 Web 可達到原生 App 所做的任何事情。而 HTML5 現所邁入的新階段，更是達到此願景的重要一步。針對即將到來的 Firefox 十周年紀念 (#Fx10)，我們已經規劃了許多有趣的活動，其主題亦將繼續推動 Web 成為創新的平台與開放的生態系統。

敬請期待後續發展。

─ Mozilla 技術長 Andreas Gal
