社交不忘隱私！

10月232012

我們非常享受能與他人串連的社交功能，而社交網絡為此大量蒐集、使用越來越多的個人資料， 但使用者常常不明瞭其中所代表的意義或影響。悍衛使用者隱私是 Mozilla 一直以來的使命，隱私是不該有任何的犧牲與妥協的，有鑑於此，我 們必需找一個兩全其美的方法，提供使用者自在又安全的瀏覽體驗。

Mozilla 持續探索將社交網路整合於 Firefox 的方式，而在最新的 Beta 測試版中, 我們開始測試為 Firefox 加入社交功能。Social API 提供開放、以 Web 技術為基礎的環境，讓使用者能藉由 Firefox 連接最喜愛的社群網站，產生一種具備社交功能、與 Firefox 高度結合如同原生、且更重要是同樣尊重我們隱私原則的體驗。率先實作 Social API 體驗的社群網站是 Facebook，而未來幾個月中還可望有更多應用登場。

Social API 其中一個要件是，在使用者想傳遞資料 (data) 給社群網站時， 才能傳遞資料。新的社交功能由您決定是否使用，在尚未前往該社群網站、並決定要啟用整合功能前，所有功能將處於停用狀態；一 旦啓用後，Firefox 會透過安全連線下載許多頁面。就像你自行瀏覽的 其他網頁一樣，透過 Social API 讀取的網頁不會有特別待遇：這些網頁可以使用 cookies 等資訊，但無法額外從 Firefox 拿取任何其他資料，或將 你的社交行為記錄傳回給 Mozilla。 舉例來說，Facebook 可以知道你開啟了對應的整合功能並下載頁面資料，就像你自己去看他們的網站一樣。

但這裡有一個細微的差異：由於現在社交內容 (social content) 將在 Firefox 介面中持續顯示，所以社群網絡可以不斷增加新東西，比如通知、狀態 更新和聊天邀請等，你不必刻意打開該社群網站的網頁。社群網絡無法因為這個新功能，得到權限獲取您瀏覽器中任何額外資訊。

我們的 Social API 中有個對隱私很棒的功能：推薦鈕。許多網站都會在頁面中放入各種分享按鈕，讓你可以將網頁分享到社群網站上；不過即使使用者沒有點選，社群網站仍可藉由這 些按鈕得知使用者去過哪些網站。我們把推薦鈕內建於 Firefox 中，在你點擊時才會將資訊送到指定的社群網站（以 Facebook 為例，就等於你「讚」了那個網頁）。一方面你仍然可以「讚」或分享網頁到社群網路上，二方面社群網 站也就沒有藉此追蹤你的機會。目前點選這個按鈕僅送出當前閱讀網頁的網址，日後可能會增加如 Open Graph（Facebook 機制中的開放社交關係資訊）標籤、標題 或圖片等資訊。

Social API 讓網絡創造一個人與人之間獨特的互動方式，利用它們自有的設計和功能，但不需犧牲掉使用者的控制權和 隱私。這只是其中的第一步！我們會繼續探索更多與社群網站整合功能的方式，同時提升使用者的選擇權、控制權和隱私。
