你的開發者工具還缺哪些功能呢？

4月162015

所有工具都源自於「如果有了這東西，我就能做得更快、更好、更輕鬆」的概念。Firefox 開發者工具就是根據這類想法而誕生，而且其中許多功能也都來自於和你一樣的 Web 開發者。

我們透過 UserVoice 討論區蒐集這些想法，並讓大家票選出其開發的優先順序。

去年四月，@hopefulcyborg 就提出應該要能「[Connect Firefox Developer Tools to [Insert Browser Here]](https://ffdevtools.uservoice.com/forums/246087-firefox-developer-tools-ideas/suggestions/5789823-connect-firefox-developer-tools-to-insert-browser)」。之後我們在十一月發表了 Firefox 開發者版本 (Developer)，其內即提供該功能的預備版本，可透過 Firefox 桌面版直接對 Chrome 與 Safari (是的，甚至可用於行動裝置) 進行除錯，而此即所謂的「Valence」。

不見得一定要多麼宏大或創新，才能算得上是好的想法與概念；重點是能更有效率的協助大家完成手上任務。以下列功能為例，都是直接取自於 UserVoice 上的反饋意見：

  • 在 Firefox 39 的檢測器 (Inspector) 中，只要拖曳 DOM 節點即可重新排序。
  • 從 Firefox 38 開始，主控台 (Console) 即可顯示更多有用的 CORS 錯誤並區隔 XHR。
  • 我們為 Firefox 37 改變了檢測器的顯示顏色，讓色彩辨識有困難的人也能更易於使用。
  • 我們為 Firefox 34 新增 Web 儲存空間的專屬檢驗面板。
  • 在 Firefox 31 時，我們將「滴管 (Eyedropper)」色彩選擇器整合至樣式檢測器之中。

某些功能可能看來沒多麼重要，但這些概念都造就了更好用、更有效率的開發者工具。這也是 Mozilla 的初衷：我們為你打造 Firefox，也要你知道我們仍虛心聆聽大家的想法。如果你對 Firefox 開發者工具有任何建議，歡迎到 UserVoice 告訴我們。如果你還沒想到任何意見，也能投票告訴我們相關建議的急迫性。相關意見都會讓負責的人看到，並協助我們打造出更好的工具給大家。

如果你想進一步參與工具的開發，我們也很樂意協助大家加入貢獻的行列。畢竟 Mozilla 本就是以開放源碼為基礎，要與社群一同提升屬於所有人的網際網路。

這裡有適合你使用的工具。努力讓工具變得更好吧！
