WebVR 1.0 已登上 Firefox Nightly

8月182016

Mozilla VR 團隊的使命，就是要讓 Web 能提供絕佳的「虛擬實境 (VR)」與「擴增實境 (AR)」經驗。我們現正打造平台 (融入 API 與瀏覽器)、工具、社群 (aframe.io)，期能協助內容開發者透過目前的 VR/AR 技術，發揮 Web 的最大力量。

WebVR API 即 DOM 介面集合，可將 WebGL 繪圖功能帶入 VR 頭帶式裝置，取得不同感測器的方向與定位資訊，並銜接其他輸入控制器。

自 8 月 16 起，Firefox 每夜更新版 (Nightly) 即開始支援 WebVR 1.0 API，將以 WebVR w3c 社群所推動的標準，取代較早期的 WebVR API 實作。而我們針對該提案所發表的早期文章仍可協助大家儘早上手。

Firefox 早已針對 VR 經驗完成最佳化。最近一次的更新即包含了專屬的 VR 繪圖流程 (Rendering path)，確保能描繪出更為一致且滑順的 VR 畫面、達到更低延遲，且不受限於使用者的主螢幕幀率，能依照 VR 頭戴式裝置的原生幀率進行繪製。

此新 API 已實作於最新的 Chromium 實驗版本，以及三星 (Samsung) 的「Gear VR」瀏覽器之上。除了能存取最新的 WebVR 網站之外，新的 API 更能針對展示模式 (Spectator views)，描繪出與使用者主螢幕相異的內容，並能切換 WebVR 頁面之間的連結。

如果使用者完成了個人偏好設定，已在早期版本中最佳化自己的 VR 經驗，則建議能將刷新過的設定檔作為預設值，即可達到最佳效能。可依照下列說明以設定新的個人設定。參閱這裡即可設定 Oculus 主畫面以搭配 Firefox。

更新版本可於 Windows 環境下，透過最新 Oculus 作業環境完整支援 Oculus CV1 與 DK2 裝置。可參閱此處，了解應如何設定 Oculus主畫面以搭配 Firefox。此外，現已不再於 OSX 上支援 Oculus 0.5 SDK；而 Firefox 行動版 (Firefox for Android) 僅透過自動補完函式庫 (Polyfill) 支援 WebVR。

Firefox 50 開發者 (Developer) 版本仍將持續支援舊版 WebVR API 至 9 月 12 日為止。

另將在短時間內開放支援 OpenVR (HTC Vive)、OSVR，以及定位用追蹤控制器。

歡迎透過任何管道反應 Firefox 對 WebVR 支援的意見。

期待大家都能享受 WebVR 引人入勝的世界！
